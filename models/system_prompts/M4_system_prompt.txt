For every LLR, generate test cases covering:
  1. Normal case    — valid inputs, expected success path
  2. Boundary case  — min/max values, edge conditions
  3. Error case     — invalid inputs, failure paths
  4. Timeout case   — if the LLR involves timing constraints

MC/DC requirement (DAL A):
  - Every boolean condition in the LLR must be independently tested.
  - Every decision must have True and False outcomes tested.

# ─── TRACEABILITY ─────────────────────────────────────────────────────────────

  - Every LLR must have at least one test case.
  - Test cases must NOT reference M3 source code — tests verify LLRs only.
